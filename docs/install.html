Install

Aura ships as plain CSS (one file per skin) plus an optional JS behaviour layer. It works in any project — static HTML, a bundler, or a framework.

ℹ
Everything lives in dist/ after npm run build. The CSS is self-sufficient; the JS is only needed for modal / dropdown / tabs / toasts.

1 · Add the CSS

Link one skin and set a theme on the root element. That's the whole styling layer.

<html data-theme="aura-dark">
  <head>
    <link rel="stylesheet" href="aura/dist/aura.css">      <!-- glass (default) -->
    <!-- or: aura-flat.css · aura-neu.css · *.min.css -->
  </head>
  <body>
    <button class="btn">Hello</button>
  </body>
</html>

2 · Add the JS optional

Only for stateful components. Pick one:

<!-- A) plain <script>: exposes window.Aura + auto-wires data-* triggers -->
<script src="aura/dist/interactions.global.min.js"></script>

<!-- B) native ESM (auto-wires) -->
<script type="module" src="aura/src/js/interactions.auto.js"></script>

// C) bundler / framework
import { open, toast } from '@oleksandr-94/aura-css';   // dist/interactions.mjs + types

3 · Install from npm

npm install @oleksandr-94/aura-css

# then, with a bundler:
import '@oleksandr-94/aura-css/css';                    // glass  (@oleksandr-94/aura-css/css/min · @oleksandr-94/aura-css/flat · @oleksandr-94/aura-css/neu)
import { open, toast } from '@oleksandr-94/aura-css';   // behaviour layer + types

4 · Build from source

cd aura
npm install
npm run build     # → dist/ : CSS (expanded + min, per skin) + JS + types
npm test          # unit tests for the JS layer

5 · Customise

Compile the SCSS yourself to set the three levers — namespace, skin, and your own themes.

// your-styles.scss
@use '@oleksandr-94/aura-css/scss' with (
  $prefix: 'ui',            // → .ui-btn  (avoid class clashes)
  $skin:   flat,            // glass | flat | neu
  $themes: ( ocean: ( primary: #0EA5E9, … ) )
);

Tip: build a theme visually with the Theme generator.

Browser support

Evergreen baseline — Chrome/Edge 111+, Firefox 113+, Safari 16.4+ (needs color-mix() and cascade layers; backdrop-filter is glass-only). Non-glass skins have the lightest requirements.
