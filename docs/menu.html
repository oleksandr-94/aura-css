Menu

A vertical navigation primitive — not a sidebar. It carries no width, position, or background of its own, so the same .menu drops cleanly into a sidebar, a drawer, or a popover. The active item rides the brand gradient.

Compact rail

Add menu--compact to collapse the menu into an icons-only rail — the markup stays exactly the same, so a sidebar can toggle between full and compact with one class. Labels and section headings hide (headings become thin dividers), the badge shrinks to a notification dot on the icon corner, and data-tip shows the item name as a tooltip on hover/focus.

The tooltip renders to the item's inline-end side, outside the rail — make sure the host sidebar doesn't clip it with overflow: hidden.

Heads up: host CSS resets

Aura components live in @layer components, so any unlayered rule in your app overrides them. Broad resets can break the menu in particular:

  • * { margin:0 } / * { padding:0 } flatten spacing on menu__section, menu__badge and item padding.
  • a { color:inherit } recolours the active menu__item (it's an <a>), so it loses its on-gradient text colour.

Scope those resets to your own containers, or put your app styles in their own @layer declared after Aura. This matters most for link-based components (menu, future breadcrumbs).
